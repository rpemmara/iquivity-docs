## Selecting an AI Model

### Choosing the Right Model

**To Change AI Model:**

1. Click **"AI Model: [Current Model]"** button
2. Browse available models
3. Review each model's features:
   - Name and description
   - Credit cost (Unlimited or specific amount)
   - Capabilities (Words, Images, etc.)
4. Select desired model
5. Click **"Apply"**

### Model Comparison

**Available Models by Provider:**

!!! info "Always Up to Date"
    All models are automatically updated to the latest versions from each provider. The specific model names may change as providers release newer versions.

#### OpenAI Models

**Latest GPT Models**
- **Type**: Text and Multimodal (text + vision)
- **Tiers Available**: Most advanced, turbo (faster), standard, lighter versions
- **Best For**: General-purpose tasks, creative writing, image analysis, code generation
- **Speed**: Varies by tier (from very fast to moderate)
- **Use When**: Need versatile, well-rounded AI performance

**Strengths:**
- Industry-leading performance
- Excellent multimodal capabilities
- Strong at following complex instructions
- Great for creative and analytical tasks

#### Anthropic Models

**Latest Claude Models**
- **Type**: Text
- **Tiers Available**: Most powerful, balanced, fastest
- **Best For**: Deep analysis, long-form content, nuanced understanding, code generation
- **Speed**: Varies by tier (from fastest to moderate)
- **Use When**: Need sophisticated reasoning and detailed responses

**Strengths:**
- Superior analytical capabilities
- Excellent for long conversations
- Nuanced understanding of context
- Strong safety and helpfulness

#### Google Models

**Latest Gemini Models**
- **Type**: Multimodal (text, images, and more)
- **Tiers Available**: Ultra (most capable), Pro (balanced), Flash (fastest)
- **Best For**: Multimodal tasks, general-purpose work, quick responses
- **Speed**: Varies by tier (from very fast to moderate)
- **Use When**: Need multimodal capabilities or Google ecosystem integration

**Strengths:**
- Strong multimodal capabilities
- Fast response times
- Good balance of quality and speed
- Integration with Google services

#### Perplexity Models

**Perplexity AI**
- **Type**: Text with built-in web search
- **Best For**: Research, current information, fact-checking, citations
- **Speed**: Fast
- **Use When**: Need real-time web data or cited sources

**Strengths:**
- Built-in real-time web search
- Provides citations and sources
- Excellent for research tasks
- Up-to-date information

#### DeepSeek Models

**DeepSeek Chat & Coder**
- **Type**: Text (with coding specialization)
- **Best For**: Programming, code generation, debugging, technical tasks
- **Speed**: Fast
- **Use When**: Working on any coding or technical projects

**Strengths:**
- Specialized for programming tasks
- Excellent code understanding
- Strong technical reasoning
- Cost-effective for development work

#### Grok (X AI) Models

**Grok**
- **Type**: Text with real-time information
- **Best For**: Conversational AI, current events, engaging responses
- **Speed**: Fast
- **Use When**: Need conversational responses or real-time information

**Strengths:**
- Engaging, personality-driven responses
- Access to real-time information
- Conversational and natural
- Good for current events

#### OpenRouter

**OpenRouter Gateway**
- **Type**: Multi-model access
- **Best For**: Exploring different models, comparing outputs
- **Speed**: Varies by selected model
- **Use When**: Want access to various models through one interface

**Strengths:**
- Access to multiple AI providers
- Regularly updated model catalog
- Flexibility to try different models
- Unified interface

### Using Prompt Suggestions

**Quick Start with Prompts:**

```
[Translate This Book]
```

**To Use:**

- AI will respond based on the selected model

**To Start a Custom Conversation:**

1. Click in the message field at the bottom

**Example Messages:**
```

"Write a professional email to decline a job offer"

"Create a workout plan for beginners"

```

**Be Specific:**

```
âŒ No Context: "Write an email"
```

## Advanced Usage Tips

### Multi-Turn Conversations

**Building Context:**

```
Turn 1: "I'm building a SaaS product for project management"
AI: [Acknowledges and asks clarifying questions]

Turn 2: "Target audience is small businesses, 10-50 employees"
AI: [Provides relevant advice]

Turn 3: "What pricing model would work best?"
AI: [Suggests pricing based on context]

AI: [Generates table with context of small business focus]
```

**Why It Works:**
- AI remembers previous context
- Builds on earlier information
- More accurate responses
- Natural conversation flow

### Combining Features

**Example Workflow:**

1. **Start with Prompt Library**
   - Select "Technical & Knowledge" prompt
   - Modify for your specific need

## Troubleshooting

### Common Issues

**Problem: AI Not Responding**
- **Cause**: Network issue or model overloaded
- **Solution**: Refresh page, try different model

**Problem: Response Quality Poor**
- **Cause**: Vague prompt or model not suited for task
- **Solution**: Be more specific in prompt, try different provider or higher-tier model

**Problem: Conversation Lost Context**
- **Cause**: Too many turns or model limitation
- **Solution**: Summarize previous context in next message

**Problem: File Upload Failed**
- **Cause**: File too large or unsupported format
- **Solution**: Compress file, convert to supported format

**Problem: Credits Running Low**
- **Cause**: Using credit-consuming models/features
- **Solution**: Switch to unlimited credit models, upgrade plan
