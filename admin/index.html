    Admin Documentation¶

    Welcome to the iQuivity Admin Panel documentation. This guide is designed for platform administrators who manage users, resellers, customers, and platform settings.

    Admin Panel Overview¶

    The Admin Panel provides comprehensive tools for managing the iQuivity platform, monitoring performance, and supporting users.

    Who This Guide Is For¶

    This documentation is for: - Super Admins - Full control over the entire system - Admins - Control over specific areas and functions - Customer Admins - Management of customer accounts

    Quick Access¶

    • Dashboard

      Monitor platform performance, revenue, users, and analytics in real-time.

      View Dashboard Guide

    • User Management

      Manage users, activities, permissions, and deletion requests.

      Manage Users

    • Announcements

      Create and manage public and dashboard announcements for users.

      Manage Announcements

    • Support Requests

      Handle user support tickets and affiliate/withdrawal requests.

      Support System

    • Reseller Management

      Manage resellers, commissions, and partnerships.

      Manage Resellers

    • Customer Management

      Create and manage company customers with admin access.

      Manage Customers

    Admin Roles & Permissions¶

    Super Admin¶

    Full Control: - Complete access to all system functions - Manage other admins - Configure system settings - Access all financial data - Manage all users, resellers, and customers

    Default Status: You are assigned as super admin by default

    Admin¶

    Limited Control: - Manage users and specific settings - Permissions limited to assigned areas - Cannot modify system-level settings - Restricted financial access

    Customer Admin¶

    Company-Level Control: - Manage company-specific settings - Control company users - Access company data only - No platform-wide permissions

    Admin Privileges¶

    Super Admins can control which areas are accessible to regular admins:

    Available Privileges:

    • ✓ Marketplace - Manage marketplace and AI models
    • ✓ Themes - Control platform themes
    • ✓ User Management - Manage platform users
    • ✓ Announcements - Create and send announcements
    • ✓ Google AdSense - Configure advertising
    • ✓ Support Requests - Handle support tickets
    • ✓ Templates - Manage content templates
    • ✓ Chat Settings - Configure chatbot settings
    • ✓ Frontend - Modify frontend settings
    • ✓ Finance - Access financial data and reports

    Key Metrics Overview¶

    Platform Performance¶

    Financial Metrics: - Total earnings and revenue sources - Referral payouts - Total spending - Cost per user analysis

    User Metrics: - New users and subscribers - Total registered users - Online users - Daily visits and AI usage

    Content Metrics: - Generated content breakdown (Text, Images, Audio, Code, Video) - Popular AI tools usage - User client distribution (Desktop vs Mobile)

    Geographic Data: - Top countries by user count - User distribution analytics

    Common Admin Tasks¶

    Daily Operations¶

    1. Monitor Dashboard - Check key metrics and alerts
    2. Review Support Requests - Address user issues
    3. Check New Users - Verify and approve new registrations
    4. Monitor Revenue - Track earnings and spending

    Weekly Operations¶

    1. User Activity Review - Analyze user engagement
    2. Process Withdrawals - Handle payout requests
    3. Review Analytics - Check performance trends
    4. Update Announcements - Communicate with users

    Monthly Operations¶

    1. Financial Reports - Generate and review reports
    2. Reseller Commissions - Process and pay commissions
    3. User Cleanup - Handle deletion requests
    4. Platform Updates - Review and plan improvements

    Best Practices¶

    Admin Best Practices

    • Monitor Daily - Check dashboard and support requests daily
    • Respond Quickly - Address support tickets within 24 hours
    • Regular Backups - Ensure data is backed up regularly
    • Security First - Use strong passwords and enable 2FA
    • Document Changes - Keep records of major configuration changes
    • Review Analytics - Weekly analysis of trends and patterns

    Security Guidelines¶

    Access Control¶

    Protecting Admin Access: - Use strong, unique passwords (minimum 8 characters) - Enable two-factor authentication (2FA) - Don't share admin credentials - Log out after each session - Use secure networks only

    User Data Protection¶

    Handling User Information: - Respect user privacy - Only access data when necessary - Process deletion requests promptly - Secure sensitive information - Comply with data regulations

    Financial Security¶

    Protecting Financial Data: - Verify withdrawal requests - Monitor unusual transaction patterns - Secure payment information - Regular financial audits - Separate financial access
